# CLAUDE.md

이 파일은 이 저장소에서 작업하는 Claude Code(claude.ai/code)를 위한 가이드입니다.

## 프로젝트 개요

마크다운(`.md`) 파일을 읽어 정적 블로그 웹사이트로 변환하는 프로젝트입니다.
글은 마크다운으로 작성하고, 이를 파싱해 깔끔하고 읽기 좋은 HTML 페이지로 렌더링합니다.

**핵심 목표**
- 마크다운 원본을 그대로 유지하면서 보기 좋은 블로그로 변환
- 읽기 경험 최우선 — 타이포그래피, 여백, 가독성 중심의 깔끔한 디자인
- 다크 모드 지원 (사용자 선택 + 시스템 설정 감지)
- 모바일 반응형 (작은 화면에서도 편안하게 읽힘)

## 기술 제약 (중요)

- **프레임워크 없음.** React, Vue, Next.js, 빌드 툴체인(webpack, Vite 등) 사용 금지.
- **순수 HTML, CSS, JavaScript(바닐라)만 사용.**
- 외부 의존성은 최소화한다. 추가가 꼭 필요하면 먼저 사용자에게 확인할 것.
  - 예외로 검토 가능한 것: 마크다운 파싱 라이브러리(예: marked.js), 코드 하이라이팅(예: highlight.js).
    단, CDN `<script>` 태그로만 포함하고 npm/번들러는 도입하지 않는다.
- npm 패키지 매니저나 `node_modules`를 도입하지 않는다. 파일은 브라우저에서 바로 열려야 한다.

## 디렉토리 구조

```
my-blog/
  index.html          # 뼈대 + 테마 FOUC 방지 인라인 스크립트, marked.js/highlight.js CDN
  css/style.css       # CSS 변수, 라이트/다크 테마, 반응형, 마크다운 요소 스타일
  js/app.js           # 해시 라우팅, fetch, 프론트매터 파싱, 렌더링, 테마 토글
  posts/posts.json    # 글 파일명 매니페스트 (배열)
  posts/*.md          # 프론트매터 포함 글 파일
  CLAUDE.md
```

## 동작 방식

- **단일 `index.html` + 해시 라우팅**: `#/`는 글 목록, `#/post/<slug>`는 글 상세.
  `<slug>`는 `.md` 확장자를 뺀 파일명이다.
- **글 발견**: 브라우저는 디렉토리를 나열할 수 없으므로 `posts/posts.json`의 파일명 배열로
  글을 찾는다. 새 글을 추가하면 반드시 이 목록에도 넣어야 한다.
- **프론트매터**: 각 `.md` 상단의 `---` 블록에서 `title`, `date`, `tags`, `summary`를 읽는다.
  `js/app.js`의 `parseFrontmatter()`가 의존성 없이 직접 파싱한다(인라인 `[a, b]`와
  하이픈 목록 모두 지원). YAML 전체 스펙이 아니라 이 필드들만 다룬다.
- **목록**은 각 글의 프론트매터를 읽어 날짜 내림차순으로 정렬해 렌더한다. `summary`가 없으면
  본문 첫 문단에서 요약을 자동 생성한다.
- **마크다운 렌더**는 marked.js, **코드 하이라이팅**은 highlight.js(둘 다 CDN)를 사용한다.

## 아키텍처 원칙

- **관심사 분리**: 구조(HTML) / 스타일(CSS) / 동작(JS)을 각각의 파일로 나눈다.
- **테마는 CSS 변수로**: 색상, 간격, 폰트 크기 등은 `:root`의 CSS 커스텀 프로퍼티로 정의하고,
  다크 모드는 이 변수들을 오버라이드하는 방식으로 구현한다. 색상 값을 컴포넌트에 하드코딩하지 않는다.
- **다크 모드**:
  - `prefers-color-scheme`로 시스템 설정을 초기값으로 감지한다.
  - 토글 버튼으로 수동 전환을 지원하고, 선택은 `localStorage`에 저장한다.
  - 새로고침 시 깜빡임(FOUC)이 없도록 테마를 가능한 한 이른 시점에 적용한다.
- **반응형**: 모바일 우선(mobile-first)으로 작성한다. 본문은 읽기 편한 최대 폭(약 65~75자,
  대략 `680px` 내외)으로 제한하고, 미디어 쿼리로 화면 크기에 맞춘다.
- **접근성**: 시맨틱 HTML(`<article>`, `<nav>`, `<main>`, `<time>` 등), 충분한 색상 대비,
  키보드 접근 가능한 토글을 유지한다.

## 디자인 방향

- **미니멀 페이퍼(paper) 톤.** 크림색 배경(`#faf6ee`)과 세리프 본문(`--font-serif`)으로
  종이책 같은 차분한 읽기 경험을 낸다. 다크 모드도 순수 검정이 아닌 따뜻한 톤의 다크 브라운
  (`#1c1812`)을 사용한다. 강조 색은 테라코타 계열(`--color-accent`) 하나만 절제해서 쓴다.
  장식보다 여백과 타이포그래피로 완성도를 낸다.
- 본문은 세리프, 메타 정보(날짜/태그)·UI 요소는 산세리프(`--font-sans`)로 구분해 위계를 준다.
- 본문은 읽기 좋은 행간(line-height ~1.7)과 폰트 크기를 사용한다.
- 코드 블록, 인용문, 목록, 이미지 등 마크다운 요소가 모두 일관되게 스타일링되어야 한다.

## 개발 / 실행

- **정적 서버가 필요하다.** `fetch`로 `.md`와 `posts.json`을 읽기 때문에 `file://`로 직접 열면
  CORS 때문에 동작하지 않는다. 정적 서버로 열 것:
  - 예: `python -m http.server 8000` 후 `http://localhost:8000` 접속, 또는 VS Code Live Server.
- 변경 후에는 데스크톱과 모바일 폭 모두에서, 그리고 라이트/다크 모드 모두에서 확인한다.

## 새 글 추가 절차

1. `posts/`에 `<slug>.md` 파일을 만들고 상단에 프론트매터를 작성한다:
   ```
   ---
   title: 글 제목
   date: 2026-07-03
   tags: [태그1, 태그2]
   summary: 목록에 보일 한 줄 요약 (생략 가능)
   ---
   ```
2. `posts/posts.json` 배열에 `"<slug>.md"`를 추가한다.
3. 홈 목록에 자동으로 날짜순으로 나타난다.

## 컨벤션

- 새 코드는 주변 코드의 스타일(들여쓰기, 네이밍, 주석 밀도)을 따른다.
- CSS 변수 이름은 의미 기반으로 짓는다 (예: `--color-bg`, `--color-text`, `--color-accent`).
- 마크다운 글 파일은 `posts/`에 모아 관리한다.
- 목록/제목/태그 등 사용자 콘텐츠를 DOM에 넣을 때는 `escapeHtml()`을 거친다(본문 마크다운은
  marked.js가 처리). 텍스트 삽입 시 이 관례를 유지한다.
