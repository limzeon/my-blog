# 오늘 배운 것

클로드 코드와 함께 이 블로그를 만들면서, 세 언어가 각자 맡은 역할을 다시 한번 명확히
이해하게 되었습니다. 오늘은 그 내용을 짧게 정리해 봅니다.

## HTML — 구조

HTML은 페이지의 **뼈대**를 담당합니다. `<header>`, `<main>`, `<article>`, `<footer>` 같은
시맨틱 태그로 콘텐츠의 의미와 위계를 표현합니다. 이 블로그에서는 `index.html` 하나가
모든 페이지의 뼈대 역할을 하고, 실제 콘텐츠는 자바스크립트가 그 안(`<main id="app">`)에
채워 넣는 방식입니다.

## CSS — 스타일

CSS는 그 뼈대에 **모양**을 입힙니다. 이번에 인상 깊었던 점은 색상이나 여백을 코드 곳곳에
흩어 쓰지 않고, `:root`에 CSS 변수(`--color-bg`, `--color-text` 등)로 모아둔 것이었습니다.

```css
:root {
  --color-bg: #ffffff;
  --color-text: #1f2328;
}

html[data-theme="dark"] {
  --color-bg: #0d1117;
  --color-text: #e6edf3;
}
```

이렇게 해두면 다크 모드는 단순히 이 변수들의 값만 바꿔주면 되고, 개별 요소마다 색을
다시 지정할 필요가 없습니다. 반응형도 마찬가지로, 미디어 쿼리 안에서 같은 변수를
다른 값으로 오버라이드하는 식으로 처리할 수 있었습니다.

## JavaScript — 동작

JavaScript는 **동작과 데이터**를 담당합니다. 이 블로그에서 자바스크립트가 하는 일은
크게 세 가지였습니다.

1. **라우팅** — 주소창의 해시(`#/`, `#/post/<slug>`)를 읽어 목록을 보여줄지 글 상세를
   보여줄지 결정합니다.
2. **데이터 가져오기** — `fetch`로 `posts.json`과 각 `.md` 파일을 읽어옵니다.
3. **렌더링** — 마크다운 프론트매터를 파싱하고, 본문을 HTML로 변환해 화면에 그립니다.

> 세 언어는 서로 대체재가 아니라 역할이 다른 도구라는 걸, 직접 나눠서 짜보니 훨씬
> 분명하게 느껴졌습니다.

## 정리

| 언어       | 역할       | 이 블로그에서 담당하는 것            |
| ---------- | ---------- | ------------------------------------- |
| HTML       | 구조       | 페이지 뼈대, 시맨틱 마크업            |
| CSS        | 스타일     | 색상·간격·반응형·다크 모드            |
| JavaScript | 동작       | 라우팅, 데이터 로딩, 렌더링           |

셋을 파일 단위로 분리해두니(`index.html`, `css/style.css`, `js/app.js`) 각자 무슨 일을
하는지 헷갈리지 않고, 수정할 때도 어디를 고쳐야 할지 바로 알 수 있어서 좋았습니다.
